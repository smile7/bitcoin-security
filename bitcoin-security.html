Bitcoin Security

    Over the past year the Bitcoin protocol proved itself to be secure enough, but not the same can be said for the wallets and services used to store and exchange cryptocurrency.

  • Encryption

    • Bitcoin uses SHA-256 encryption. Essential for the security of the bitcoin protocol is the transaction blockchain. The blockchain consists of multiple ‘blocks’ which store transaction history. In the beginning of the blockchain is the initial block, also known as the genesis block. The new blocks contain transactions and solved hashes and placed after the initial block, thus creating a blockchain. Bitcoins can only be spent after a verified transaction.

  • Double Spending

    • Although there is a discussion on this point, in fact, it is possible to double-spend the same amount of Bitcoins, for example in the following cases:

    1. Sending two conflict transactions by an attacker if the merchant doesn’t wait for confirmation of the transaction
    2. Pre-mining one transaction into a block and spend the same amount of coins before releasing this block into the blockchain

      3. This is rarely seen as the computing power required for this process to succeed is too much compared to the legitimately mining Bitcoins.
